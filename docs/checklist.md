# Build Checklist — MacroDay

## Build Preferences

- **Build mode:** Step-by-step (one item per `/build` session; run `/clear` between each item)
- **Comprehension checks:** Yes — a quick question after each step to confirm the piece makes sense
- **Verification:** Yes — per-item; learner runs the app and confirms what they see before moving on
- **Git:** Commit after each item with message: `Complete step N: [title]`
- **Check-in cadence:** Balanced — brief explanation of what was built and why, then verify and move on
